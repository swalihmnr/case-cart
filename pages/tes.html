🛒 E-commerce Shopping Cart System

Complete Code Implementation Planner

📋 Project Overview

Database: MongoDB (NoSQL)
Backend: Node.js + Express
Timeline: 4-6 Weeks
Architecture: RESTful API

Phase 1: Project Setup & Configuration

Week 1 - Foundation

1Environment Setup

  • Initialize Node.js project with npm init
  • Install core dependencies: express, mongoose, dotenv, bcrypt, jsonwebtoken
  • Setup MongoDB Atlas or local MongoDB instance
  • Create .env file with configuration variables
  • Setup folder structure
src/ ├── models/ ├── controllers/ ├── routes/ ├── middleware/ ├── config/ └── utils/

2Database Connection

  • Create config/database.js for MongoDB connection
  • Implement connection error handling
  • Test database connectivity

Phase 2: Database Schema Design

Week 1-2 - Data Models

1User Model (models/User.js)

  • Basic info: email, password, firstName, lastName, phone
  • Avatar/profile image URL
  • Role field (customer, admin, vendor)
  • Embedded addresses array with shipping/billing types
  • Wishlist array (product IDs)
  • Timestamps (createdAt, updatedAt, lastLogin)
  • Account status flags (emailVerified, isActive)
Indexes: Create unique index on email and index on role
  • Implement hashPassword() pre-save hook
  • Implement comparePassword() method
  • Implement generateAuthToken() method

2Product Model (models/Product.js)

  • Basic info: name, slug, description, SKU
  • Category reference (ObjectId)
  • Brand field
  • Price object (regular, sale, currency)
  • Inventory tracking with quantity and thresholds
  • Images array with URL, alt text, primary flag
  • Specifications object (flexible key-value)
  • Variants array for product options
  • Ratings object (average, count)
  • Tags array for search functionality
  • SEO metadata object
  • Status field (active/inactive/draft)
Indexes: slug (unique), category, status, text search on name/description/tags

3Category Model (models/Category.js)

  • Name, slug, description fields
  • Parent category reference for hierarchy
  • Image URL for category display
  • Order field for sorting
  • Active status flag
  • Subcategories array

4Cart Model (models/Cart.js)

  • User ID reference (null for guest carts)
  • Session ID for guest users
  • Items array with product details, quantity, price
  • Totals object (subtotal, tax, shipping, discount, total)
  • Coupon information if applied
  • Shipping method selection
  • Expiry timestamp for guest carts

5Order Model (models/Order.js)

  • Order number (unique identifier)
  • User reference
  • Items array from cart
  • Shipping and billing addresses
  • Payment information (status, method, transaction ID)
  • Order status tracking
  • Pricing breakdown (subtotal, tax, shipping, discount)
  • Timestamps for all status changes

Phase 4: Core API Endpoints

Week 3-4 - Business Logic

1Product APIs

  • GET /api/products - List products with pagination/filtering
  • GET /api/products/:id - Get single product
  • POST /api/products - Create product (admin only)
  • PUT /api/products/:id - Update product (admin only)
  • DELETE /api/products/:id - Delete product (admin only)
  • GET /api/products/search - Search products
  • GET /api/products/category/:categoryId - Products by category

2Cart APIs

  • GET /api/cart - Get user's cart
  • POST /api/cart/items - Add item to cart
  • PUT /api/cart/items/:itemId - Update item quantity
  • DELETE /api/cart/items/:itemId - Remove item from cart
  • DELETE /api/cart - Clear entire cart
  • POST /api/cart/apply-coupon - Apply discount coupon
  • POST /api/cart/merge - Merge guest cart with user cart

3Order APIs

  • POST /api/orders - Create order from cart
  • GET /api/orders - Get user's order history
  • GET /api/orders/:id - Get single order details
  • PUT /api/orders/:id/cancel - Cancel order
  • PUT /api/orders/:id/status - Update order status (admin)

4User Profile APIs

  • GET /api/users/profile - Get user profile
  • PUT /api/users/profile - Update profile
  • POST /api/users/addresses - Add address
  • PUT /api/users/addresses/:id - Update address
  • DELETE /api/users/addresses/:id - Delete address
  • GET /api/users/wishlist - Get wishlist
  • POST /api/users/wishlist/:productId - Add to wishlist
